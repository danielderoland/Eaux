Gradient Boosting — LightGBM

Oui, c'est du machine learning. Le modèle apprend tout seul à combiner l'heure, le jour, la saison et le passé récent pour prédire le débit — sans qu'on lui explique les règles.

C'est quoi le machine learning ici ?

L'idée est de transformer le problème en une question classique : étant donné ce qu'on sait à l'instant t (heure, jour, mois, valeurs passées…), quel sera le débit ?

On donne au modèle des milliers d'exemples passés — chaque ligne est un instant t avec toutes ses informations et la vraie valeur du débit observée. Le modèle apprend à reproduire ces valeurs, puis generalise sur de nouveaux instants.

Chaque instant devient une ligne. Le modèle apprend à prédire la colonne orange à partir des grises.

LightGBM capture-t-il les cycles ?

Oui — mais de façon implicite. C'est la différence fondamentale avec Prophet.

Prophet décompose explicitement la série : il nomme et affiche chaque composante (tendance, cycle journalier, cycle hebdo, cycle annuel). LightGBM, lui, ne les nomme jamais — il apprend les mêmes effets en se servant des features qui encodent ces cycles.

Chaque cycle de Prophet correspond à un groupe de features dans LightGBM. Le résultat appris est le même, la représentation est différente.

C'est quoi les lags ?

Un lag est simplement la valeur du débit à un instant passé. "Lag 24h" = le débit mesuré exactement 24 heures avant l'instant qu'on veut prédire.

Intuition

Si à 8h hier il y avait 95 m³/h, il y a de bonnes chances qu'à 8h aujourd'hui on soit proche de cette valeur — sauf événement exceptionnel. Le lag capture cette inertie de la consommation.

6 lags sont utilisés : 5 min, 15 min, 1h, 6h, 24h et 7 jours en arrière.

Comment mesurer la qualité : MAE et MAPE

Pour savoir si le modèle est bon, on compare ses prédictions aux vraies valeurs sur le jeu de test (les 3 derniers mois, non vus pendant l'entraînement). Deux métriques résument cette erreur :

MAE — Mean Absolute Error
2,81
Erreur absolue moyenne. En moyenne, le modèle se trompe de 2,81 m³/h par prédiction.

Si le débit réel est 90 m³/h et qu'on prédit 87,2 m³/h, l'erreur est 2,8 m³/h — c'est ça que mesure la MAE.

Facile à interpréter car exprimé dans la même unité que le débit.
MAPE — Mean Absolute Percentage Error
4,12%
Erreur relative moyenne. En moyenne, on se trompe de 4,12% du débit réel.

Si le débit réel est 90 m³/h, une erreur de 4,12% représente ± 3,7 m³/h. Si le débit est 50 m³/h, c'est ± 2,1 m³/h.

Utile pour comparer des PR de tailles différentes (un grand PR a des débits absolus bien plus élevés qu'un petit).
Comment interpréter 4,12% sur de l'eau ?

En hydraulique urbaine, un MAPE en dessous de 5–8% est généralement considéré comme bon pour un modèle basé sur des données historiques. En dessous de 3% est excellent. Ces chiffres sont sur données simulées (propres) — sur tes vraies données avec des pannes capteur et anomalies, compte plutôt 6–12%.

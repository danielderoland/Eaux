Réseaux de neurones — LSTM & Transformers temporels

L'approche la plus puissante sur le papier, mais souvent surdimensionnée pour des séries temporelles très cycliques comme la consommation d'eau. Ici non implémentée — voici pourquoi et comment.

Verdict pour ce projet

Sur des données d'eau urbaine à 3 PR avec des patterns très réguliers, le deep learning apportera vraisemblablement un gain marginal (−1 à −2% de MAPE) au prix d'une complexité bien supérieure : GPU recommandé, hyperparamètres nombreux, temps d'entraînement long, modèle opaque.

À utiliser seulement si LightGBM et Prophet ont été épuisés et qu'il reste un gap de performance significatif à combler.

Principe : lire la séquence brute

Au lieu de construire des features manuellement (heure, sin/cos…), le deep learning reçoit directement la séquence brute des valeurs passées et apprend tout seul à détecter les motifs utiles.

Chaque cellule LSTM transmet à la suivante un "état de mémoire" — le réseau retient le contexte des dernières heures pour prédire le pas suivant.

Les 3 architectures possibles

LSTM

Long Short-Term Memory. Cellules récurrentes avec portes d'oubli/entrée. Robuste, entraînable sur CPU. Point de départ recommandé.

GRU

Version simplifiée du LSTM (moins de paramètres). Souvent aussi bon, plus rapide à entraîner.

TFT

État de l'art actuel. Interprétable (attention heads). Nécessite GPU et beaucoup de données.

# pip install torch ou pip install tensorflow import torch import torch.nn as nn class LSTMDebit(nn.Module): def __init__(self, input_size=1, hidden=64, layers=2): super().__init__() self.lstm = nn.LSTM(input_size, hidden, layers, batch_first=True, dropout=0.2) self.fc = nn.Linear(hidden, 1) def forward(self, x): out, _ = self.lstm(x) return self.fc(out[:, -1, :]) # dernière cellule → prédiction # Entrée : fenêtres glissantes de 288 pas (24h) → prédire le pas suivant # X.shape = (N_samples, 288, 1) y.shape = (N_samples, 1)

La vraie difficulté du LSTM n'est pas le code — c'est la création des fenêtres glissantes, le choix de la taille de fenêtre (24h ? 48h ? 7j ?), la normalisation des données, et le réglage du learning rate scheduler. Pour l'eau urbaine, commence par LightGBM et Prophet — ils feront 95% du chemin avec 5% de la complexité.
